STAGE 03 · WEEK 07 — 09 平台化

LLMOps 应用平台可视化

构建 LLMOps 可视化管理平台，实现 Prompt 模板管理、流式响应展示与用户授权认证体系，让 AI 应用可管理、可监控。

Jinja2 SSE JWT RBAC Prompt Engineering
OVERVIEW

阶段概述

当 ChatBot 具备记忆、RAG 与 Agent 能力后，下一个挑战是"可管理性"。本阶段将构建一个可视化的 LLMOps 管理平台，让 Prompt、模型响应与用户权限都成为可控的资产。

第七周聚焦 Prompt 模块——设计模板数据模型、变量注入与版本管理，支持 A/B 测试与可视化编辑；第八周深入响应模块——SSE 流式协议、Redis 响应缓存、多模型切换与降级、监控仪表盘。

第九周搭建授权认证体系——用户注册登录、JWT 双 Token 机制、RBAC 权限模型与安全加固。最终交付一个具备 Prompt 管理 + 流式响应 + 认证的 LLMOps 可视化管理平台。

LEARNING OBJECTIVES

学习目标

设计模板系统，支持版本控制、变量注入与 A/B 测试。

实现 SSE 端点与多模型动态切换、降级策略。

认证与 RBAC 权限

搭建 JWT 双 Token 认证与角色权限控制体系。

可视化管理后台

完成平台核心管理功能

交付 LLMOps 平台核心管理能力，使 AI 应用可管理、可监控、可运营。

TECHNICAL HIGHLIGHTS

技术要点

Jinja2 模板引擎

使用 Jinja2 实现 Prompt 变量注入与渲染，支持系统变量与自定义变量校验。

SSE 协议

基于 text/event-stream 实现服务端推送，配合心跳保活与断点续传。

JWT 双 Token 认证

Access Token 短时效存内存，Refresh Token 长时效存 HttpOnly Cookie，自动续签。

RBAC 权限模型

角色-权限映射实现 API 级与路由级双重控制，支撑多租户场景。

FAQ

常见问题

Q1

SSE 连接经常断开？

实现 Last-Event-ID 机制支持断点续传，添加心跳包（每 15s 发送注释行），Nginx 配置 proxy_buffering off 避免缓冲堆积。

Q2

JWT Token 存哪里？

Access Token 存内存（前端 Vuex / Pinia），Refresh Token 存 HttpOnly Cookie，避免 XSS 攻击窃取。

Q3

Prompt 版本管理怎么做？

每次修改创建新版本，保留历史版本，支持 diff 对比与一键回滚，配合 A/B 测试用数据驱动迭代。
