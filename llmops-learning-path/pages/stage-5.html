OVERVIEW

阶段概述

本阶段聚焦多 LLM 接入、统计模块与生产部署，将前四个阶段构建的 LLMOps 平台从开发环境推向生产可用。第 13 章完成多模型管理器与本地 LLM（Ollama）接入，第 14 章补齐 Token 统计、WebApp 发布与项目优化部署，系统性提升平台可用性与交付标准。

在多 LLM 层面，通过 YAML 配置 + 动态导入设计模型管理器，支持 GPT、Ollama 等供应商快速切换与 function calling 扩展；在统计层面，分别统计输入 / 输出 Token 并接入 ECharts 可视化面板；在发布层面，打通 WebApp 凭证、会话与对话接口，并接入 markdown-it 渲染 AI 消息。

最后通过 404/403 错误页面、Vue hooks 优化、TypeScript 类型补全与前后端 Dockerfile 容器化，将项目交付到生产环境，构建一个稳定运行、可观测、可快速迭代的生产级 AI 平台。

FOR YOU

与你经验的对照

可迁移的技能
  • 数据模型设计 → 供应商/模型实体结构与数据仓库建模思路一致
  • 统计监控 → Token 统计与 ECharts 面板，和 BI 报表可视化直接对口
  • 云平台部署 → 你有阿里云/华为云经验，Docker 与 Nginx 概念不陌生
需要补课的内容
  • Docker 容器化 → 简历无 Docker 实操，Dockerfile 与镜像构建是重点
  • 动态导入 + YAML 配置驱动 → Python 设计模式，需要补动态导入机制
  • WebApp 发布链路 → 凭证、会话、对话接口的前后端联动是新增内容
学习建议
  • 第 13 章重点：模型管理器的"配置驱动"思想，可类比数据仓库的元数据管理
  • 第 14 章按"统计 → 发布 → 优化"三节推进，每节都有独立 API 文档可对照
  • Docker 学习：先跑通 Vue 与 API 两个 Dockerfile，再理解多服务编排

FAQ

常见问题

Q

模型管理器如何支持新供应商？

A:

在 YAML 配置文件中新增供应商条目，利用动态导入按配置项加载对应服务类，pydantic 校验初始化参数，无需修改核心代码即可接入新供应商。

Q

WebApp 凭证失效如何重生成？

A:

调用发布模块的凭证重生成接口，旧凭证立即失效并写入新凭证标识，前端 WebApp 通过新凭证获取基础信息与对话接口。

Q

前后端 Dockerfile 有何区别？

A:

Vue 前端使用多阶段构建，先 node 构建静态资源再用 nginx 托管；API 后端基于 python 镜像安装依赖并启动 Flask 服务，两者可独立构建与部署。
